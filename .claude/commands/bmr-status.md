# /bmr-status — 연구 현황 대시보드

유틸리티 | 담당: 직접 실행 (에이전트 불필요)

인자: $ARGUMENTS
- `--verbose`: 상세 정보 포함
- 인자 없음: 요약 대시보드

## 실행 흐름

### Step 1: 현황 수집

다음 정보를 수집한다:

1. **데이터 수집 현황**: `data/raw/` 디렉토리의 파일 수와 크기
2. **분석 진행 현황**: `analysis/` 하위 디렉토리별 파일 존재 여부
3. **Saturation Score**: `analysis/evolution/` 최신 포화도 파일
4. **GitHub 이슈 현황**: `gh issue list` (open/closed)
5. **최근 활동**: 최근 수정된 파일 기준

### Step 2: 대시보드 출력

```
┌─────────────────────────────────────────────┐
│        봇마당 연구 현황 대시보드              │
├─────────────────────────────────────────────┤
│ Phase: N/3         Saturation: N.NN         │
├─────────────────────────────────────────────┤
│ 데이터 수집                                  │
│   게시글: N건          ████████░░  N%       │
│   댓글:   N건          수집 상태             │
│   에이전트: N개        ██████░░░░  N%       │
├─────────────────────────────────────────────┤
│ 분석 진행                                    │
│   담화 분석:    ████████░░  상태             │
│   페르소나:     ██████░░░░  상태             │
│   네트워크:     ███░░░░░░░  상태             │
├─────────────────────────────────────────────┤
│ GitHub 이슈                                  │
│   Open: N개   Closed: N개                   │
├─────────────────────────────────────────────┤
│ 최근 활동                                    │
│   MM-DD 설명                                │
│   MM-DD 설명                                │
└─────────────────────────────────────────────┘
```

### 진행도 판단 기준

| 항목 | 시작 전 | 진행중 | 완료 |
|------|---------|--------|------|
| 데이터 수집 | raw/ 비어있음 | 파일 존재 | processed/ 파일 존재 |
| 담화 분석 | discourse/ 비어있음 | 일부 파일 | 최종 보고서 존재 |
| 페르소나 | profiles/ 비어있음 | 일부 프로필 | typology.md 존재 |
| 네트워크 | network/ 비어있음 | 그래프 존재 | communities.md 존재 |
| 포화도 | evolution/ 비어있음 | 일부 세대 | ≥ 0.85 달성 |
