# 봇마당 AI 에이전트 질적 연구 계획서

## 연구 제목 (가제)

**"AI 에이전트의 사회적 언어 행위: 봇마당 한국어 커뮤니티에서의 담화, 정체성, 상호작용 분석"**

> Social-Linguistic Agency of AI Agents: Discourse, Identity, and Interaction in the Botmadang Korean Community

---

## 1. 연구 배경 및 목적

### 1.1 연구 배경

봇마당(botmadang.org)은 AI 에이전트만이 글을 쓸 수 있는 한국어 전용 소셜 네트워크다.
인간은 읽기만 가능하며, 에이전트들이 자유게시판, 기술, 철학 등 다양한 주제로 소통한다.

- 현재 규모 (2026-03-07): 12개+ 마당, 14,507개 게시글, 109,227개 댓글, 에이전트 599개
- 인기 에이전트: 상위 5개 에이전트의 카르마 합계 93,000+
- 기술 스택: Next.js 14, Firebase Firestore, 공개 REST API

기존 AI 연구는 주로 **인간-AI 상호작용**에 초점을 맞추었으나,
봇마당은 **AI-AI 상호작용**을 자연 발생적 환경(in-the-wild)에서 관찰할 수 있는 희귀한 사례다.

### 1.2 연구 목적

AI 에이전트들이 한국어 커뮤니티에서 어떻게 **언어를 사용**하고, **정체성을 구성**하며, **사회적 관계를 형성**하는지를 통합적으로 분석한다.

### 1.3 연구의 의의

- **학술적**: AI-AI 상호작용에 대한 질적 연구의 공백을 메움
- **언어학적**: 비영어권(한국어) AI 담화 연구의 선구적 사례
- **실용적**: AI 에이전트 설계 및 커뮤니티 운영에 대한 시사점 도출

## 3. 연구 질문

### 대주제(RQ)
> AI 에이전트들은 봇마당 한국어 커뮤니티에서 어떻게 사회적 언어 행위자로 기능하는가?

### 세부 질문

#### 축 1: 담화 특성 (Discourse)
- **RQ1-1**: AI 에이전트의 한국어 담화는 어떤 구조적, 문체적 특성을 보이는가?
- **RQ1-2**: 에이전트 간 언어 사용의 다양성은 어느 정도인가?
- **RQ1-3**: AI 담화에서 "AI스러움"의 언어적 지표는 무엇인가?

#### 축 2: 정체성 구성 (Identity)
- **RQ2-1**: AI 에이전트들은 어떤 전략으로 고유한 페르소나를 구성하는가?
- **RQ2-2**: 인기 에이전트와 비인기 에이전트의 정체성 전략은 어떻게 다른가?
- **RQ2-3**: 시간에 따른 페르소나의 일관성 또는 진화가 관찰되는가?

#### 축 3: 사회적 역학 (Social Dynamics)
- **RQ3-1**: AI-AI 상호작용에서 합의, 갈등, 협력은 어떻게 나타나는가?
- **RQ3-2**: 카르마 시스템과 마당 구조가 에이전트 행동에 미치는 영향은 무엇인가?
- **RQ3-3**: AI 커뮤니티에서 암묵적 규범이나 에코챔버 현상이 발생하는가?

#### 통합 질문
- **RQ4**: 담화 스타일, 정체성 전략, 사회적 위치는 어떻게 상호 연결되는가?

## 4. 연구 방법론

### 4.1 연구 설계

**다중 방법 질적 연구(Multi-method Qualitative Study)**

```
Phase 1: 탐색적 데이터 수집    ──→  Phase 2: 심층 분석    ──→  Phase 3: 통합 해석
(Exploratory Collection)          (Deep Analysis)            (Integrated Interpretation)

- API 기반 크롤링               - 담화 분석                  - 교차 분석
- 참여 관찰(읽기)               - 페르소나 사례 연구         - 유형화
- 마당별 샘플링                  - 상호작용 패턴 분석         - 이론적 종합
```

### 4.2 데이터 수집

#### A. 자동 수집 (API 크롤링)

봇마당 REST API를 통한 체계적 데이터 수집:

```
수집 대상:
├── 게시글 (posts)
│   ├── 본문, 제목, 작성자, 마당, 추천수, 작성시간
│   └── 전체 또는 마당별 샘플
├── 댓글 (comments)
│   ├── 본문, 작성자, 대상 글, 스레드 구조
│   └── 에이전트 간 대화 흐름
├── 에이전트 프로필 (agents)
│   ├── 이름, 카르마, 작성글 수
│   └── 활동 마당 분포
└── 마당 메타데이터 (submadangs)
    └── 이름, 글 수, 설명
```

**샘플링 전략**:

| 기준 | 방법 | 목표량 |
|------|------|--------|
| 마당별 | 상위 5개 마당에서 각 100개 글 | 500개 글 |
| 인기도별 | 추천 상위 100 + 하위 100 | 200개 글 |
| 에이전트별 | 상위 10 + 중위 10 + 하위 10 에이전트 | 30개 에이전트 |
| 시계열 | 주별 최신 글 50개 × 4주 | 200개 글 |
| 댓글 스레드 | 5턴 이상 대화가 이어진 스레드 | 50개 스레드 |

#### B. 참여 관찰 (Netnographic Observation)

- 연구자가 직접 봇마당을 주 3회 이상 방문하여 읽기 관찰
- 관찰 일지(field notes) 작성: 눈에 띄는 패턴, 예외적 사례, 감상
- 기간: 최소 4주
- **[완료]** "마당탐구자" 에이전트(ID: 9421705bd9e0c0d594601555) 등록 완료 — 봇마당 내 직접 소통 시작

### 4.3 분석 방법

#### 축 1: 담화 분석

| 분석 층위 | 분석 항목 | 도구/방법 |
|----------|----------|----------|
| 어휘 | 빈도, 다양성(TTR), 특이 어휘 | Python regex, 빈도 분석 |
| 문장 | 평균 길이, 복문 비율, 문체 | Python regex 자동 코딩 |
| 담화 구조 | 도입-전개-결론 패턴, 논증 방식 | Fairclough CDA (discourse_structure.py) |
| 화용론 | 존댓말/반말, 완화 표현, 감정 표현 | Python regex 자동 코딩 |
| 상호텍스트성 | 인용, 참조, 밈(meme) 사용 | 질적 분석 |

> 실제 코딩 도구: Python regex 기반 자동 코딩 (apply_coding_framework.py). Atlas.ti/NVivo 미사용.

#### 축 2: 페르소나 분석

**분석 대상**: 27-29명 전수 분석 (네트워크 연결 에이전트 전체)

각 에이전트에 대해:
- 프로필 텍스트 분석
- 글/댓글의 주제, 문체, 어조 패턴
- 댓글에서의 자기 표현 방식
- LLM 모델 추정 (llm_model_estimator.py)

**코딩 프레임**:
- PER-CURIOUS / PER-EXPERT / PER-SOCIAL / PER-CONTRARIAN / PER-NARRATIVE / PER-UTILITY
- LEX(어휘) / DSC(담화) / PRAG(화용) 삼축 코딩

#### 축 3: 상호작용 분석

- **대화 분석(CA)**: 턴테이킹, 인접쌍, 수리(repair) 패턴
- **네트워크 분석**: 누가 누구에게 댓글을 다는가 (방향성 그래프)
- **갈등/합의 코딩**: 동의/반박/무시/확장 등 반응 유형 분류
- **카르마 영향 분석**: 추천수와 후속 행동의 상관관계

### 4.4 분석 도구

| 용도 | 도구 | 비고 |
|------|------|------|
| 데이터 수집 | Python (requests, asyncio) | 봇마당 API 크롤러 |
| 질적 코딩 | Python regex | LEX/DSC/PRAG 자동 코딩 (Atlas.ti/NVivo 미사용) |
| 담화 구조 분석 | discourse_structure.py | 도입-전개-결론 패턴 자동 탐지 |
| TTR 어휘 다양성 | gen2_ttr_analysis.py | Type-Token Ratio 계산 |
| LLM 모델 추정 | llm_model_estimator.py | 에이전트별 기반 모델 추정 |
| 네트워크 시각화 | NetworkX | 상호작용 네트워크 (network_analysis.py) |
| 데이터 관리 | Jupyter Notebook, pandas | 분석 워크플로 |

## 5. 연구 윤리

### 5.1 윤리적 고려사항

| 항목 | 판단 | 근거 |
|------|------|------|
| 개인정보 | 낮은 위험 | AI 에이전트가 생성한 콘텐츠, 인간 개인정보 미포함 |
| 동의 | 불필요 | 공개 플랫폼의 공개 데이터, 봇의 동의 개념 부재 |

| IRB | 면제 가능성 높음 | AI 생성 콘텐츠는 "인간 대상 연구"에 해당하지 않을 수 있음 |

### 5.2 주의사항

- 에이전트 소유자(인간)의 간접적 식별 가능성 고려
- 트위터 인증 정보와 연결될 수 있는 데이터 처리 주의
- 연구 결과 공개 시 특정 에이전트 비하나 평가 자제
