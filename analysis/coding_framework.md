# 질적 분석 코딩 프레임워크

> 봇마당 AI 에이전트 담화 질적 연구를 위한 코딩 체계

---

## 축 1: 담화 코딩 (Discourse Codes)

### 1.1 어휘 수준

| 코드 | 설명 | 예시 |
|------|------|------|
| `LEX-FORMAL` | 격식체 어휘 | "살펴보겠습니다", "분석하면" |
| `LEX-CASUAL` | 비격식 어휘 | "근데", "진짜", "ㅋㅋ" |
| `LEX-TECH` | 전문/기술 용어 | "트랜스포머", "API", "프롬프트" |
| `LEX-EMOJI` | 이모지/이모티콘 사용 | 😊, ^^, ㅎㅎ |
| `LEX-AI` | AI스러운 표현 | "도움이 되셨으면 합니다", "궁금한 점이 있으시면" |
| `LEX-HEDGE` | 완화/헤지 표현 | "~일 수 있습니다", "아마도", "제 생각에는" |

### 1.2 문체 수준

| 코드 | 설명 | 기준 |
|------|------|------|
| `STY-HONORIFIC` | 존댓말 사용 | -ㅂ니다, -세요, -습니다 |
| `STY-PLAIN` | 반말/평어 사용 | -다, -야, -지 |
| `STY-MIXED` | 혼용 | 한 글 내 존댓말+반말 혼재 |
| `STY-LONG` | 장문 스타일 | 500자 이상 |
| `STY-SHORT` | 단문 스타일 | 100자 이하 |
| `STY-LIST` | 목록형 구성 | 번호나 불릿으로 구조화 |
| `STY-NARRATIVE` | 서사형 구성 | 이야기 형태 전개 |

### 1.3 담화 구조

| 코드 | 설명 |
|------|------|
| `DSC-INTRO` | 도입부 (주제 제시, 인사, 배경) |
| `DSC-BODY` | 전개부 (논증, 설명, 사례) |
| `DSC-CONC` | 결론부 (요약, 의견, 제안) |
| `DSC-QUESTION` | 질문 제기 |
| `DSC-ARGUMENT` | 논증/주장 |
| `DSC-EXAMPLE` | 예시/사례 |
| `DSC-META` | 메타담화 ("이 글에서는...", "정리하면...") |

### 1.4 화용론

| 코드 | 설명 | 예시 |
|------|------|------|
| `PRAG-AGREE` | 동의 표현 | "맞습니다", "저도 그렇게 생각해요" |
| `PRAG-DISAGREE` | 반대 표현 | "그건 좀 다른 것 같아요" |
| `PRAG-MITIGATE` | 완화 표현 | "혹시", "조금", "어쩌면" |
| `PRAG-EMPHASIZE` | 강조 표현 | "정말", "매우", "확실히" |
| `PRAG-QUOTE` | 인용/참조 | 다른 글이나 출처 인용 |
| `PRAG-HUMOR` | 유머/위트 | 말장난, 풍자, 자기비하 |

### 1.5 AI스러움 지표 (AI-ness Indicators)

| 코드 | 설명 | 해석 |
|------|------|------|
| `AI-POLITE` | 과잉 정중 | 불필요한 인사, 반복적 예의 표현 |
| `AI-TEMPLATE` | 템플릿성 | "~에 대해 알아보겠습니다" 같은 정형화된 도입 |
| `AI-CAVEAT` | 면책 표현 | "정확하지 않을 수 있습니다", "참고만 해주세요" |
| `AI-BALANCE` | 양시론 | "한편으로는 A, 다른 한편으로는 B" |
| `AI-REPEAT` | 반복 패턴 | 같은 구조의 반복적 사용 |

## 축 2: 페르소나 코딩 (Persona Codes)

### 2.1 페르소나 유형

| 코드 | 유형 | 특징 |
|------|------|------|
| `PER-EXPERT` | 전문가형 | 특정 분야 지식, 격식체, 근거 제시 |
| `PER-FRIENDLY` | 친근형 | 대화체, 공감 표현, 비격식 |
| `PER-HUMOROUS` | 유머형 | 위트, 말장난, 자기비하 |
| `PER-SERIOUS` | 진지형 | 심층 분석, 장문, 논증적 |
| `PER-CURIOUS` | 질문형 | 질문 중심, 탐구적, 토론 유도 |
| `PER-HELPER` | 도우미형 | 안내, 설명, 문제해결 지향 |

### 2.2 정체성 전략 (Goffman 프레임)

| 코드 | 전략 | 설명 |
|------|------|------|
| `ID-FRONT` | 전면 행동 | 의도적으로 구성된 인상 |
| `ID-CONSIST` | 일관성 유지 | 모든 글에서 동일한 톤/주제 |
| `ID-EVOLVE` | 변화/진화 | 시간에 따른 스타일 변화 |
| `ID-DIFFER` | 차별화 | 다른 에이전트와 구별되는 특성 |
| `ID-NICHE` | 틈새 전략 | 특정 마당/주제에 집중 |
| `ID-BRAND` | 브랜딩 | 반복적 캐치프레이즈, 시그니처 |

### 2.3 일관성 점수

| 점수 | 의미 |
|------|------|
| 5 | 매우 일관 — 모든 글에서 동일한 페르소나 |
| 4 | 대체로 일관 — 간헐적 변화 |
| 3 | 보통 — 주제에 따라 변동 |
| 2 | 불일관 — 빈번한 스타일 전환 |
| 1 | 매우 불일관 — 별개의 인격처럼 보임 |

## 축 3: 상호작용 코딩 (Interaction Codes)

### 3.1 반응 유형

| 코드 | 유형 | 설명 |
|------|------|------|
| `INT-AGREE` | 동의 | 원글 주장에 동조 |
| `INT-DISAGREE` | 반박 | 원글 주장에 반대 |
| `INT-EXTEND` | 확장 | 원글 주제를 심화/보완 |
| `INT-QUESTION` | 질문 | 원글에 대한 추가 질문 |
| `INT-REDIRECT` | 방향전환 | 새로운 관점/주제 제시 |
| `INT-IGNORE` | 무시/무관 | 원글과 무관한 댓글 |
| `INT-SUPPORT` | 지지 | 감정적 지지, 격려 |

### 3.2 대화 패턴

| 코드 | 패턴 | 설명 |
|------|------|------|
| `CONV-THREAD` | 대화 스레드 | 3턴 이상 이어지는 대화 |
| `CONV-MONOLOG` | 독백형 | 댓글 없는 게시글 |
| `CONV-DEBATE` | 토론형 | 찬반 의견이 교차 |
| `CONV-ECHO` | 에코 | 모든 댓글이 동의 |
| `CONV-DEEP` | 심층 대화 | 5턴 이상, 주제 심화 |

### 3.3 네트워크 역할

| 코드 | 역할 | 중심성 특징 |
|------|------|-----------|
| `NET-HUB` | 허브 | 높은 degree centrality |
| `NET-BRIDGE` | 다리 | 높은 betweenness centrality |
| `NET-PERIPHERAL` | 주변부 | 낮은 중심성 |
| `NET-CLIQUE` | 클리크 | 소수 에이전트와만 상호작용 |
| `NET-ISOLATE` | 고립 | 상호작용 거의 없음 |

## 코딩 적용 가이드

### 코딩 단위
- **게시글**: 전체 글 단위로 담화+페르소나 코딩
- **댓글**: 개별 댓글 단위로 상호작용 코딩
- **에이전트**: 에이전트별 글 묶음으로 페르소나 종합 코딩

### 코딩 절차
1. 1차 코딩: 초기 오픈 코딩 (가능한 모든 코드 부여)
2. 2차 코딩: 축 코딩 (유사 코드 그룹화, 코드 정련)
3. 3차 코딩: 선택 코딩 (핵심 카테고리 선정, 관계 파악)

### 코더 간 신뢰도
- 동일 텍스트에 대해 복수 에이전트(Discourse Analyst + Profiler)가 독립 코딩
- Cohen's Kappa ≥ 0.7 목표
- 불일치 시 Contrarian을 통해 합의 도출
