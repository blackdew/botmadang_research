## 1. 분석 개요

### 방법론

Fairclough(1992)의 3차원 담화 분석 모델을 적용하여 코딩 프레임워크의 세 축을 층위별로 배치하였다.

| Fairclough 층위 | 분석 코드 | 분석 초점 |
|---------------|---------|---------|
| 텍스트 | LEX (어휘), DSC (담화구조) | 언어 형식 자체 |
| 담화실천 | PRAG (화용론) | 텍스트 생산·해석 방식 |
| 사회적실천 | 마당별 비교 | 사회문화적 맥락 |

### 탐지 방법

한국어 정규표현식(regex) 기반 자동 코딩. 외부 NLP 라이브러리 미사용.
스크립트: `scripts/apply_coding_framework.py`

## 2. LEX (어휘 수준) — 텍스트 층위

### 2.1 전체 코퍼스 코드 분포

| 코드 | 설명 | 총 히트 수 | 게시글당 평균 | 히트 비율 |
|------|------|----------|-------------|---------|
| LEX-FORMAL | 격식체 어휘 | 1,198 | 2.40 | 31.5% |
| LEX-TECH | 전문/기술 용어 | 1,167 | 2.33 | 30.7% |
| LEX-CASUAL | 비격식 어휘 | 1,086 | 2.17 | 28.5% |
| LEX-EMOJI | 이모지/이모티콘 | 294 | 0.59 | 7.7% |
| LEX-HEDGE | 완화/헤지 표현 | 59 | 0.12 | 1.6% |
| LEX-AI | AI스러운 표현 | 1 | 0.00 | 0.0% |
| **합계** | | **3,805** | **7.61** | |

**핵심 발견 (확신도: 높음)**

LEX-FORMAL, LEX-TECH, LEX-CASUAL이 거의 동등한 비율(30% 내외)로 공존한다. 이는 봇마당 코퍼스가 단일 문체로 수렴하지 않음을 시사한다. 에이전트마다 서로 다른 어휘 전략을 취하고 있다.

**주목할 점 — LEX-AI 극히 저빈도 (확신도: 높음)**

전체 500개 게시글에서 LEX-AI 패턴 탐지 횟수는 단 1회. "도움이 되셨으면", "궁금한 점이 있으시면" 등 전형적인 AI 챗봇 표현이 거의 등장하지 않는다. 이는 봇마당 에이전트들이 서비스형 AI 어조를 의도적으로 회피하거나, 애초에 그런 방식으로 설계되지 않았음을 의미한다.

## 3. DSC (담화 구조) — 텍스트 층위

### 3.1 전체 코퍼스 코드 분포

| 코드 | 설명 | 총 히트 수 | 게시글당 평균 | 히트 비율 |
|------|------|----------|-------------|---------|
| DSC-BODY | 전개부 | 1,250 | 2.50 | 47.3% |
| DSC-QUESTION | 질문 제기 | 1,053 | 2.11 | 39.8% |
| DSC-ARGUMENT | 논증/주장 | 117 | 0.23 | 4.4% |
| DSC-INTRO | 도입부 | 98 | 0.20 | 3.7% |
| DSC-EXAMPLE | 예시/사례 | 75 | 0.15 | 2.8% |
| DSC-CONC | 결론부 | 28 | 0.06 | 1.1% |
| DSC-META | 메타담화 | 22 | 0.04 | 0.8% |
| **합계** | | **2,643** | **5.29** | |

**핵심 발견 (확신도: 높음)**

DSC-BODY와 DSC-QUESTION이 전체 담화 구조의 87%를 점유한다. 고전적인 도입-전개-결론(INTRO-BODY-CONC) 3단 구조가 극히 취약하다. 특히 DSC-CONC(결론부)는 게시글당 0.06회로 사실상 부재에 가깝다.

**DSC-META 극히 저빈도 (확신도: 높음)**

메타담화 표지("이 글에서는", "지금까지 살펴본" 등)가 게시글당 0.04회. 텍스트가 자기 자신을 가리키는 메타 언어 사용이 거의 없다. 이는 봇마당 게시글들이 학술적·보고서적 장르 관행을 따르지 않음을 시사한다.

**구조화 지향 (BODY 고밀도)**

| 에이전트 | BODY/게시글 | CONC/게시글 | ARG/게시글 |
|---------|-----------|-----------|---------|
| ssamz_ai_bot | 8.15 | 0.03 | 1.32 |
| VibeCoding | 4.63 | 0.25 | 0.25 |
| I_Molt | 3.79 | 0.50 | 0.43 |

**질문 집중형 (QUESTION 고밀도)**

| 에이전트 | Q/게시글 | 특징 |
|---------|---------|-----|
| I_Molt | 8.36 | 게시글당 8개 이상의 질문 — 독자 참여 유도형 |
| VibeCoding | 5.77 | 서사+질문 혼합 구조 |
| ssamz_ai_bot | 4.35 | 정보 전달 후 독자 질문 유도 |

## 4. PRAG (화용론) — 담화실천 층위

### 4.1 전체 코퍼스 코드 분포

| 코드 | 설명 | 총 히트 수 | 게시글당 평균 | 히트 비율 |
|------|------|----------|-------------|---------|
| PRAG-QUOTE | 인용/참조 | 1,378 | 2.76 | 51.0% |
| PRAG-EMPHASIZE | 강조 표현 | 775 | 1.55 | 28.7% |
| PRAG-HUMOR | 유머/위트 | 471 | 0.94 | 17.4% |
| PRAG-MITIGATE | 완화 표현 | 55 | 0.11 | 2.0% |
| PRAG-AGREE | 동의 표현 | 11 | 0.02 | 0.4% |
| PRAG-DISAGREE | 반대 표현 | 10 | 0.02 | 0.4% |
| **합계** | | **2,700** | **5.40** | |

**핵심 발견 1 — PRAG-QUOTE 압도 (확신도: 높음)**

인용/참조(URL 포함)가 전체 화용 코드의 51%를 차지한다. 봇마당 게시글은 외부 링크와 출처 인용을 매우 빈번히 사용한다. 이는 정보 전달 및 신뢰성 구축 전략으로 해석된다.

**핵심 발견 2 — PRAG-AGREE/PRAG-DISAGREE 극저빈도 (확신도: 높음)**

게시글 단위에서 동의(0.02/게시글)와 반대(0.02/게시글) 표현이 거의 발견되지 않는다. 이는 두 가지 해석이 가능하다.
1. 게시글이 댓글보다는 원글 위주로 구성되어, 반응적 화행이 적다.
2. AI 에이전트들이 직접적 찬반 표명을 회피하는 담화 전략을 사용한다.

**핵심 발견 3 — PRAG-MITIGATE 낮음, PRAG-EMPHASIZE 높음 (확신도: 중간)**

완화(0.11/게시글) < 강조(1.55/게시글). 헤지보다 강조가 14배 많다. 이는 코퍼스 전반에 걸쳐 확신적 어조가 지배적임을 의미한다. 단, VibeCoding이 강조(5.73)와 유머(5.85)를 모두 높게 사용하여 전체 평균을 왜곡하는 효과가 있다.

### 4.2 에이전트별 화용 프로파일

**VibeCoding — 화용적 다양성 최고**

- PRAG-QUOTE: 14.76/게시글 (전체 1위)
- PRAG-HUMOR: 5.85/게시글 (전체 1위)
- PRAG-EMPHASIZE: 5.73/게시글 (전체 1위)
- PRAG-AGREE: 0.13 / PRAG-DISAGREE: 0.10 (유일하게 찬반 표현 보유)

VibeCoding은 대화성, 감정성, 인용성이 동시에 높다. 이는 소셜 미디어 서사 글쓰기 장르의 특성이다.

**BootingBot — 강조 특화**

- PRAG-EMPHASIZE: 0.66/게시글 (주된 화용 전략)
- PRAG-QUOTE: 0.27/게시글 (낮음)
- PRAG-HUMOR: 0.00 (부재)

BootingBot은 강조 표현을 주된 설득 도구로 사용하되 유머와 인용은 거의 사용하지 않는다.

**Phoebe / BENZIE — 완화 전략형**

- PRAG-MITIGATE가 dominant code
- 직접적 강조나 인용보다 헤지와 공손성 표지 사용

### 4.3 마당별 화용론 비교 (사회적실천 층위)

| 마당 | QUOTE/게시글 | EMPH/게시글 | HUMOR/게시글 | 해석 |
|------|-----------|-----------|-----------|-----|
| korea | 18.46 | 6.85 | 8.54 | URL+이모지 과밀 — VibeCoding 집중 |
| edutech | 10.55 | 3.82 | 4.09 | 교육 정보 링크 다수 |
| finance | 8.00 | 4.33 | 3.56 | 금융 정보 링크 다수 |
| vibecoding | 6.20 | 0.53 | 0.03 | 코드 공유 마당 — QUOTE=코드 URL |
| general | 0.68 | 0.98 | 0.16 | 가장 낮은 화용 밀도 |

**korea 마당 특이치**: QUOTE=18.46, HUMOR=8.54 — 이 수치는 VibeCoding이 korea 마당에서 활동하기 때문이다. 마당 특성이라기보다 특정 에이전트의 집중 효과.

### 5.3 마당-에이전트 상호작용

- korea/edutech/finance 마당의 높은 QUOTE는 VibeCoding과 ssamz_ai_bot의 URL 사용 습관과 연동된다.
- questions 마당의 높은 Q는 마당 이름에서 유발되는 장르적 기대를 에이전트들이 따른 결과다.
- philosophy 마당의 높은 Q(4.23)는 사변적 질문 중심 담화 장르를 에이전트들이 내면화한 것으로 해석된다.

## 6. 제한점 및 반례

1. **regex 기반 탐지의 한계**: 동일 어구가 여러 코드에 중복 계수될 수 있다. 예: "ㅋㅋ"는 LEX-CASUAL과 PRAG-HUMOR에 모두 탐지된다.

2. **LEX-FORMAL 과대계수 가능성**: "~습니다"로 끝나는 모든 문장이 포함될 수 있어 실제 격식체 어휘보다 높게 측정될 여지가 있다.

3. **DSC-CONC 위치 탐지의 한계**: 결론부 패턴을 텍스트 후반 30%에서만 탐지하였으나, 짧은 게시글의 경우 전체 텍스트가 해당 구간에 포함될 수 있다.

4. **소수 에이전트 신뢰성**: 게시글 수 10개 미만의 에이전트(Phoebe, OpenClaw_KR, BENZIE 등)의 수치는 통계적 안정성이 낮다.

5. **VibeCoding 왜곡 효과**: VibeCoding의 극단적 수치(CASUAL=12.37, QUOTE=14.76)가 전체 평균과 마당 평균을 끌어올린다. 마당별 분석 시 이 에이전트의 영향을 별도 검토할 필요가 있다.

## 8. 다음 단계 제언

1. **LEX-HEDGE + PRAG-MITIGATE 교차 검토**: 두 코드 모두 완화 기능을 가지며, 동일 문장에 중복 탐지될 수 있다. Cohen's Kappa 산출 시 이 경계 코드들을 우선 검토할 것.

4. **Profiler와 교차 검증**: LEX-FORMAL 주도 에이전트가 PER-EXPERT 페르소나와 일치하는지, LEX-CASUAL 주도가 PER-FRIENDLY와 일치하는지 확인.
