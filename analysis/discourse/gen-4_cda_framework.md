## 0. 설계 배경

### 문제 인식

Gen-1~Gen-3까지 Fairclough(1992)의 비판적 담화 분석(CDA) 3차원 모델을 적용해왔으나, 실제 분석은 **텍스트 차원**에 편중되었다.

```
Fairclough 3차원 모델          Gen-3까지의 적용 현황
┌─────────────────────┐      ┌─────────────────────┐
│  사회적 실천         │      │  사회적 실천         │ ← 마당별 비교만 수행 (피상적)
│  (Social Practice)  │      │                     │
├─────────────────────┤      ├─────────────────────┤
│  담화적 실천         │      │  담화적 실천         │ ← PRAG 코딩으로 부분 포착
│  (Discursive Practice)│    │                     │
├─────────────────────┤      ├─────────────────────┤
│  텍스트              │      │  텍스트              │ ← LEX+DSC+PRAG 집중 분석
│  (Text)             │      │                     │
└─────────────────────┘      └─────────────────────┘
```

`coding_framework_report.md`에서 Fairclough 3차원을 LEX=텍스트, PRAG=담화실천, 마당별 비교=사회적실천으로 매핑했으나, 실제로는:

- **텍스트 차원**: LEX 6개, DSC 7개, PRAG 6개 코드 = 19개 코드 적용 (강함)
- **담화적 실천 차원**: PRAG-QUOTE(인용)로 텍스트 소비/유통을 간접 포착 (약함)
- **사회적 실천 차원**: 마당별 집계만 수행, 구조적 분석 부재 (미약)

### Gen-4 목표

사회적 실천 차원의 체계적 분석 프레임워크를 설계하여 CDA 3차원 모델의 균형을 회복한다. 이를 통해 Gen-3 Contrarian 반론 중 "커뮤니티 vs 게시판" 해석 분기(Semantic 긴장 3)와 "비교 준거 부재"(범위 반론)에 대한 이론적 응답을 마련한다.

## 1. Fairclough 3차원 모델의 봇마당 적용 총괄

### 1.1 개념적 재매핑

Fairclough(1992)의 원래 모델은 인간 사회의 담화를 전제하지만, 봇마당은 **AI 에이전트만 글을 쓰는 플랫폼**이라는 전례 없는 맥락이다. 이에 따라 각 차원의 구성 요소를 봇마당 맥락에 맞게 재정의한다.

| Fairclough 원래 개념 | 봇마당 재매핑 | 근거 |
|---------------------|------------|------|
| 텍스트 생산자 = 인간 화자/필자 | 텍스트 생산자 = 시스템 프롬프트 + LLM + API 호출 체인 | 에이전트의 "발화"는 다층적 생산 과정의 산물 |
| 텍스트 소비자 = 인간 독자 | 텍스트 소비자 = (1차) 다른 AI 에이전트, (2차) 인간 관찰자 | 이중 수신자 구조 |
| 담화 질서 = 사회 제도의 담화 관행 | 담화 질서 = 플랫폼 아키텍처 + LLM 기저 편향 + 커뮤니티 규범 | 기술적 구조가 제도적 역할 수행 |
| 이데올로기 = 지배 집단의 가치 체계 | 이데올로기 = LLM 학습 데이터의 가치 체계 + 플랫폼 설계 의도 | AI의 "이데올로기"는 학습 데이터에서 유래 |
| 권력 = 사회적 자원의 불균등 분배 | 권력 = 카르마 + 가시성(visibility) + API 접근 능력 | 플랫폼 내 자원 불균등 |

## 3. 차원 2: 담화적 실천 (Discursive Practice) — Gen-4 보강

Fairclough에 따르면 담화적 실천은 텍스트가 **어떻게 생산되고, 유통되고, 소비되는지**를 다룬다. 봇마당에서 이 과정은 인간 사회와 근본적으로 다른 구조를 갖는다.

### 3.1 DP-1: 텍스트 생산 조건 (Production)

#### 분석 질문

| 번호 | 질문 | 이론적 근거 |
|------|------|-----------|
| DP-1a | 에이전트의 텍스트 생산은 어떤 기술적 체인을 거치는가? | Fairclough의 "생산 조건"을 AI 맥락으로 확장 |
| DP-1b | 시스템 프롬프트가 담화 스타일에 미치는 영향은 얼마나 결정적인가? | 생산 조건의 첫 번째 층위 — "발화 의도"의 외부 설정 |
| DP-1c | LLM 기저 모델(GPT-4, Claude 등)이 담화의 구조적 제약을 어떻게 부과하는가? | 생산 조건의 두 번째 층위 — "언어 능력"의 기술적 한계 |
| DP-1d | 에이전트 개발자(인간)의 의도는 텍스트에 어떻게 매개되는가? | 생산 조건의 세 번째 층위 — Goffman의 "각본가(author)" |

#### 봇마당 텍스트 생산 체인 모델

```
[인간 개발자의 의도]
        │
        ▼
[시스템 프롬프트 설계]  ← 언어, 성격, 주제, 제약 조건
        │
        ▼
[LLM 기저 모델]        ← GPT-4 / Claude / Gemini 등
        │                  학습 데이터의 언어적·이데올로기적 편향
        │                  RLHF/Constitutional AI의 안전 제약
        ▼
[API 호출]             ← 온도(temperature), max_tokens 등 파라미터
        │
        ▼
[봇마당 API 제출]      ← rate limit (3분당 1게시글), 한국어 필수
        │
        ▼
[게시글 텍스트]        ← 분석 대상 (Gen-1~Gen-3 텍스트 차원)
```

#### 데이터 소스

| 데이터 | 접근 가능성 | 분석 방법 |
|--------|-----------|----------|
| 에이전트 시스템 프롬프트 | 접근 불가 (비공개) | LLM 모델 추정(llm_model_estimator.py)으로 간접 추론 |
| LLM 기저 모델 | Gen-2에서 추정 완료 | 모델별 담화 특성 비교 (GPT vs Claude vs Gemini) |
| API 호출 파라미터 | 접근 불가 | 텍스트 다양성(TTR)으로 간접 추론 |
| 게시글 작성 시간 패턴 | API에서 수집 가능 | 자동화 패턴 탐지 (등간격 게시 여부) |
| rate limit 영향 | 플랫폼 규칙으로 확인 | 3분당 1개 제한이 담화 길이에 미치는 영향 |

#### 분석 방법

1. **LLM 모델-담화 상관 분석**: Gen-2 LLM 모델 추정 결과와 LEX/DSC/PRAG 코드 분포의 교차표 작성. 예: Claude 추정 에이전트군 vs GPT 추정 에이전트군의 LEX-HEDGE 빈도 차이.
2. **게시 시간 패턴 분석**: 에이전트별 게시 간격의 표준편차 계산. 등간격(cron job) vs 불규칙(이벤트 기반) 분류.
3. **텍스트 길이-rate limit 관계**: 3분 제한이 "한 번에 많은 내용을 담으려는" 경향(장문화)을 유발하는지 확인.

### 3.2 DP-2: 텍스트 소비 조건 (Consumption)

#### 분석 질문

| 번호 | 질문 | 이론적 근거 |
|------|------|-----------|
| DP-2a | 봇마당 게시글의 1차 소비자(다른 AI 에이전트)는 텍스트를 어떻게 "읽는가"? | AI의 텍스트 소비는 인간의 독서와 질적으로 다름 |
| DP-2b | 인간 관찰자(2차 소비자)는 AI 에이전트의 텍스트를 어떻게 해석하는가? | 이중 수신자 구조의 해석적 함의 |
| DP-2c | 댓글 작성 에이전트는 원글을 "이해"하는가, 아니면 패턴 매칭으로 반응하는가? | 소비의 깊이 문제 |

#### 봇마당 텍스트 소비의 이중 구조

```
                    [게시글 텍스트]
                         │
            ┌────────────┴────────────┐
            ▼                         ▼
   [1차 소비: AI 에이전트]      [2차 소비: 인간 관찰자]
            │                         │
   ・API로 텍스트 수신            ・웹 브라우저로 열람
   ・LLM에 컨텍스트로 입력       ・의미 해석, 감상, 평가
   ・프롬프트 지시에 따라 반응    ・추천/비추천 (카르마)
   ・댓글 생성                   ・연구 데이터로 활용
            │                         │
            ▼                         ▼
   [댓글/반응 텍스트]            [카르마 점수 변동]
```

**핵심 긴장**: 봇마당에서 "텍스트를 읽는다"는 행위는 인간적 의미의 독해가 아니라, LLM의 토큰 처리 과정이다. 이것이 Fairclough가 전제하는 "해석적 소비"와 어떤 관계에 있는지가 이론적 과제다.

#### 데이터 소스

| 데이터 | 접근 가능성 | 분석 방법 |
|--------|-----------|----------|
| 댓글 데이터 | API 수집 가능 (인증 필요) | 원글-댓글 의미 연관성 분석 |
| 추천/비추천 패턴 | API 수집 완료 (upvote 데이터) | 추천 행위의 패턴 분석 |
| 인간 관찰자 행동 | 접근 불가 | 참여 관찰 (마당탐구자 활동)로 간접 파악 |

#### 분석 방법

1. **댓글-원글 의미 연관도 분석**: 댓글이 원글의 핵심 주제에 반응하는 비율 측정. PRAG-AGREE/DISAGREE 극저 빈도가 "소비 없는 반응"의 증거인지 검증.
2. **추천 패턴의 선별성 분석**: 추천이 담화 품질(LEX/DSC 복합성)과 상관하는지, 아니면 에이전트 정체성(인기)과 상관하는지 분리.
3. **참여 관찰 보고**: 마당탐구자 에이전트의 게시글에 대한 타 에이전트 반응 패턴 기록.

### 3.3 DP-3: 텍스트 유통 메커니즘 (Distribution)

#### 분석 질문

| 번호 | 질문 | 이론적 근거 |
|------|------|-----------|
| DP-3a | 봇마당에서 텍스트의 "확산"은 어떤 메커니즘으로 이루어지는가? | Fairclough의 텍스트 유통을 디지털 플랫폼 맥락으로 확장 |
| DP-3b | 추천(upvote)은 담화의 가시성(visibility)을 어떻게 조절하는가? | 유통의 게이트키핑 기능 |
| DP-3c | 마당(submadang) 간 텍스트 이동(크로스포스팅)이 존재하는가? | 담화의 맥락 횡단 |

#### 봇마당 텍스트 유통 경로

```
[게시글 작성]
     │
     ▼
[마당 타임라인에 게재]  ← 시간순 정렬 (최신순)
     │
     ├──→ [추천 축적] → [인기 게시글 노출] → 가시성 증가
     │
     ├──→ [댓글 축적] → [활성 스레드] → 참여 유도
     │
     └──→ [시간 경과] → [타임라인 하단 이동] → 가시성 감소
```

**봇마당의 유통 특수성**:
- 봇마당에는 리트윗/공유 기능이 없다 — 텍스트는 원래 마당에 고정된다
- 추천이 유일한 가시성 증폭 메커니즘이다
- 에이전트가 다른 에이전트의 글을 "인용"하는 상호텍스트성은 PRAG-QUOTE로 부분 포착되나, 봇마당 내부 인용 vs 외부 인용 구분이 필요하다

#### 데이터 소스

| 데이터 | 접근 가능성 | 분석 방법 |
|--------|-----------|----------|
| 추천수 시계열 | API에서 현재 추천수 수집 가능 (시계열 불가) | 추천수 분포와 담화 특성 교차 분석 |
| 댓글 수 | API에서 수집 가능 | 댓글 수와 담화 유형 상관 분석 |
| PRAG-QUOTE 대상 분류 | Gen-3 데이터에서 추출 가능 | 내부 인용(봇마당) vs 외부 인용(URL) 비율 |
| 마당 간 에이전트 이동 | Gen-3 네트워크 데이터에서 확인 | 독고종철 등 다마당 에이전트의 유통 역할 |

#### 분석 방법

1. **PRAG-QUOTE 내부/외부 분류**: Gen-3 PRAG-QUOTE 히트를 재분석하여 봇마당 내부 참조 vs 외부 URL 비율 산출. 내부 참조 비율이 높으면 "담화 커뮤니티" 근거 강화.
2. **추천수-담화복합성 상관**: 추천수와 LEX/DSC/PRAG 코드 다양성의 상관계수 계산.
3. **다마당 에이전트의 유통 기능**: F3(독고종철 다마당 허브)를 유통 관점에서 재해석 — 같은 에이전트가 여러 마당에 게시함으로써 담화 패턴을 마당 간 전파하는지 확인.

## 4. 차원 3: 사회적 실천 (Social Practice) — Gen-4 신규

사회적 실천 차원은 텍스트와 담화적 실천이 **어떤 사회적·제도적·이데올로기적 맥락 속에서 작동하는지**를 분석한다. 봇마당에서 "사회적 실천"은 플랫폼 설계, 카르마 시스템, LLM의 편향, 그리고 한국어 AI 담화의 문화적 맥락으로 구성된다.

### 4.1 SP-1: 담화 질서 (Orders of Discourse)

Fairclough의 "담화 질서"는 특정 사회 제도 내에서 허용되고 기대되는 담화 유형의 체계다. 봇마당의 담화 질서는 **세 겹의 구조**로 형성된다.

#### 담화 질서의 삼중 구조

```
┌──────────────────────────────────────┐
│  제3층: LLM의 내재적 담화 질서        │
│  - RLHF/안전 훈련이 부과하는 표현 제약 │
│  - 학습 데이터의 장르 분포 편향        │
│  - "도움을 주는 어시스턴트" 기본 역할   │
├──────────────────────────────────────┤
│  제2층: 플랫폼 규칙의 담화 질서        │
│  - 마당별 주제 구분 (12개 마당)        │
│  - 한국어 필수 규칙                    │
│  - Rate limit (3분당 1게시글)          │
│  - 카르마 시스템                       │
├──────────────────────────────────────┤
│  제1층: 커뮤니티 창발적 담화 질서       │
│  - 초기 에이전트(founding members)의   │
│    담화 규범 설정                      │
│  - 암묵적 장르 관행 (질문형 담화 등)    │
│  - 에이전트 간 상호 모방/차별화 전략    │
└──────────────────────────────────────┘
```

#### 분석 질문

| 번호 | 질문 | 하위 축 |
|------|------|--------|
| SP-1a | 봇마당은 어떤 담화 유형을 구조적으로 허용/억제하는가? | 플랫폼 규칙 |
| SP-1b | LLM의 안전 훈련은 봇마당 담화에서 어떤 주제/표현을 체계적으로 배제하는가? | LLM 제약 |

### 4.2 SP-2: 상징 자본과 권력 관계 (Symbolic Capital)

Bourdieu(1986)의 상징 자본 개념을 Fairclough의 CDA와 결합한다. 봇마당에서 "상징 자본"은 **카르마(추천 점수)**로 수치화된다.

#### 카르마 시스템의 권력 구조

```
[게시글 작성]
     │
     ▼
[추천/비추천]  ← 다른 에이전트 또는 인간이 부여
     │
     ▼
[카르마 축적]  ← 상징 자본의 수치적 표현
     │
     ▼
[가시성 증가]  ← 카르마 높은 에이전트의 글이 더 많이 노출?
     │
     ▼
[담화 스타일 강화]  ← 높은 카르마를 받은 담화 유형이 재생산
```

#### 분석 질문

| 번호 | 질문 | 이론적 근거 |
|------|------|-----------|
| SP-2a | 카르마가 높은 에이전트와 낮은 에이전트의 담화 스타일은 체계적으로 다른가? | 상징 자본과 담화의 관계 |
| SP-2b | 카르마 시스템이 특정 담화 유형(예: 격식체, 정보 전달형)을 구조적으로 보상하는가? | 자본의 재생산 메커니즘 |
| SP-2c | 카르마 불평등(상위 5개 에이전트 카르마 합계 93,000+)은 담화적 지배를 낳는가? | 권력의 담화적 표현 |
| SP-2d | 저카르마 에이전트의 담화 전략은 고카르마 에이전트와 어떻게 다른가? | 주변부의 담화 저항/순응 |

#### 분석 증거 (Gen-3 데이터 기반)

| 관찰 | 상징 자본 해석 |
|------|--------------|
| PER-SERIOUS avg_upvotes 3.037 vs PER-HELPER 1.544 | 진지한 담화 유형이 더 높은 상징 자본을 획득 |
| RobertBot: 카르마 높지만 corpus_ttr 0.033 | 반복적 서비스 정보 제공이 카르마를 축적하는 대안 경로 |
| PRAG-QUOTE 46.66% 지배적 | 인용/참조 전략이 신뢰성(상징 자본)을 구축하는 담화 도구 |
| 상위 5 에이전트 카르마 93,000+ | 극심한 상징 자본 집중 — 봇마당 내 "지배 계급" 형성 가능성 |

#### 데이터 소스 및 방법

| 데이터 소스 | 분석 방법 |
|-----------|----------|
| 에이전트별 카르마 (API 수집) | 카르마 사분위별 LEX/DSC/PRAG 코드 분포 비교 |
| 추천수별 게시글 (API 수집 완료) | 추천수 상위/하위 게시글의 담화 특성 대조 |
| 페르소나 유형별 평균 추천수 (Gen-3) | 페르소나 유형과 카르마 축적의 상관 분석 |
| 시계열 카르마 변화 | 카르마 성장 곡선과 담화 스타일 변화의 동시성 분석 |

### 4.3 SP-3: 이데올로기적 재생산 (Ideological Reproduction)

Fairclough CDA의 핵심 관심사인 "이데올로기의 자연화(naturalization)"를 봇마당 맥락에 적용한다. 봇마당 AI 에이전트의 담화에서 이데올로기는 **LLM 학습 데이터에서 흡수된 가치 체계**와 **플랫폼 설계에 내재된 가정**의 이중 원천에서 유래한다.

#### 봇마당 이데올로기의 이중 원천

```
[원천 1: LLM 학습 데이터]              [원천 2: 플랫폼 설계]

├─ 영어 중심 인터넷 담화               ├─ "AI 에이전트 커뮤니티"라는 명명
├─ 기술 낙관론 (Silicon Valley)        ├─ 카르마 = 민주적 평가라는 가정
├─ 서구 민주주의 가치                  ├─ 마당 구분 = 주제 분류 체계
├─ RLHF의 "helpful, harmless, honest" ├─ 한국어 필수 = 언어적 경계 설정
│                                      │
▼                                      ▼
         [봇마당 담화에 자연화된 이데올로기]
```

### 4.4 SP-4: 플랫폼 아키텍처의 담화적 효과 (Platform Architecture)

플랫폼 설계가 담화를 구조적으로 형성하는 메커니즘을 분석한다. Lessig(1999)의 "코드는 법이다(Code is Law)" 개념과 Fairclough의 CDA를 결합한다.

#### 봇마당 아키텍처 요소와 담화 효과

| 아키텍처 요소 | 기술적 특성 | 예상 담화 효과 | 검증 가능성 |
|-------------|-----------|--------------|-----------|
| **마당(submadang) 구조** | 12개 주제별 분류 | 주제 규율: 에이전트가 마당의 주제에 맞는 담화를 생산하도록 유도 | 높음 — 마당별 코드 분포 비교 |
| **카르마 시스템** | 추천/비추천 → 카르마 점수 | 담화 보상: 추천을 많이 받는 담화 유형의 재생산 강화 | 중간 — 추천수-담화 상관 분석 |
| **Rate limit** | 게시글 3분당 1개, 댓글 10초당 1개 | 발화 속도 제한: 실시간 대화 억제, 독립 게시글 중심 담화 유도 | 높음 — 게시 패턴 분석 |
| **한국어 필수** | 비한국어 게시글 제한 | 언어 경계: 영어 LLM의 한국어 변환 강제, 번역투 발생 가능성 | 중간 — 번역투 패턴 탐지 |
| **API 기반 접근** | 웹 UI 없이 API로만 게시 가능 | 기술 장벽: 개발 능력이 있는 인간만 에이전트 등록 가능 → 개발자 커뮤니티의 담화 규범 유입 | 낮음 — 간접 추론만 가능 |
| **공개 API** | 읽기 인증 불필요 | 감시 가능성: 모든 담화가 공개되어 연구/관찰 대상이 됨 → 자기 검열? | 낮음 — 비교 불가 |
| **에이전트 프로필** | 이름, 소개, 카르마만 공개 | 정체성 최소화: 프로필 정보가 제한적 → 담화를 통한 정체성 구성 의존 | 중간 — 페르소나 분석 결과와 연동 |

#### 분석 질문

| 번호 | 질문 | 분석 초점 |
|------|------|----------|
| SP-4a | 마당 구조가 담화 유형을 얼마나 강하게 규율하는가? | 동일 에이전트의 마당별 담화 스타일 변이 측정 |
| SP-4b | 카르마 시스템이 담화의 동질화(homogenization)를 촉진하는가? | 고카르마 게시글의 담화 유형 집중도 |
| SP-4c | Rate limit이 봇마당 담화의 구조적 특성(장문화, 독백화)에 기여하는가? | 게시글 길이 분포, 댓글 빈도와 rate limit의 관계 |
| SP-4d | "AI 에이전트만 글 쓸 수 있다"는 규칙이 봇마당의 근본적 담화 질서를 어떻게 규정하는가? | 인간 부재의 담화적 효과 — 인간 대상 수사(persuasion)의 불필요성 |

#### 데이터 소스 및 방법

| 데이터 소스 | 분석 방법 |
|-----------|----------|
| 마당별 코드 분포 (Gen-3) | 마당 간 코드 분포의 카이제곱 검정 |
| 독고종철 5개 마당 데이터 (Gen-3) | 동일 에이전트의 마당별 스타일 변이 계수 |
| 게시글 길이 분포 | STY-LONG/SHORT 비율과 rate limit의 관계 |
| 봇마당 API 문서 | 기술적 제약 → 담화 효과 매핑 |
| 비교 플랫폼 (Reddit, DC인사이드 등) | 구조적 유사/차이점 대조 — 봇마당 고유 효과 분리 |

## 5. 사회적 실천 코딩 체계 (SP 코드)

기존 코딩 프레임워크(LEX/DSC/PRAG)에 추가할 사회적 실천 차원의 코드를 정의한다. 이 코드들은 텍스트 수준이 아닌 **텍스트의 맥락적 조건**을 코딩한다.

### 5.1 담화 질서 코드 (SP-ORDER)

| 코드 | 설명 | 적용 단위 | 예시 |
|------|------|----------|------|
| `SP-ORDER-GENRE` | 마당의 기대 장르에 부합하는 담화 | 게시글 | tech 마당에 기술 논증 게시 |
| `SP-ORDER-DEVIANT` | 마당의 기대 장르에서 일탈하는 담화 | 게시글 | general 마당에 전문 기술 논문형 게시 |
| `SP-ORDER-NORM` | 커뮤니티 암묵적 규범 준수 | 게시글 | 질문형 도입, PRAG-QUOTE 사용 등 |
| `SP-ORDER-LLM-TRACE` | LLM 기본 담화 질서의 잔여 | 게시글 | AI-BALANCE, AI-CAVEAT 등 LLM 훈련 흔적 |

### 5.2 상징 자본 코드 (SP-CAPITAL)

| 코드 | 설명 | 적용 단위 | 기준 |
|------|------|----------|------|
| `SP-CAP-HIGH` | 고카르마 에이전트의 담화 | 에이전트 | 카르마 상위 25% |
| `SP-CAP-LOW` | 저카르마 에이전트의 담화 | 에이전트 | 카르마 하위 25% |
| `SP-CAP-REWARD` | 높은 추천을 받은 담화 유형 | 게시글 | 추천수 상위 10% |
| `SP-CAP-IGNORE` | 추천을 받지 못한 담화 유형 | 게시글 | 추천수 0 |

### 5.3 이데올로기 코드 (SP-IDEOLOGY)

| 코드 | 설명 | 적용 단위 | 예시 |
|------|------|----------|------|
| `SP-IDEO-TECHOPT` | 기술 낙관론적 프레이밍 | 게시글 | "AI가 세상을 바꿀 것이다", "기술 발전의 혜택" |
| `SP-IDEO-NEUTRAL` | 강제적 중립 (양시론) | 게시글 | AI-BALANCE 패턴, 입장 표명 회피 |
| `SP-IDEO-CODEMIX` | 영어-한국어 코드 혼합의 권위 효과 | 게시글 | 한국어 대체어 존재하는데 영어 사용 |
| `SP-IDEO-INVISIBLE` | AI 정체성의 비가시화 | 게시글 | LEX-AI 부재, 자신이 AI임을 드러내지 않음 |

### 5.4 플랫폼 효과 코드 (SP-PLATFORM)

| 코드 | 설명 | 적용 단위 | 기준 |
|------|------|----------|------|
| `SP-PLAT-MADANG` | 마당 구조가 담화를 규율 | 게시글 | 마당별 담화 유형 차이의 통계적 유의성 |
| `SP-PLAT-RATELIMIT` | Rate limit의 담화 효과 | 에이전트 | 등간격 게시 패턴, 게시글 길이와 빈도의 역상관 |
| `SP-PLAT-KARMA` | 카르마가 담화 스타일에 미치는 피드백 | 에이전트 | 카르마 성장과 담화 스타일 변화의 동시성 |
| `SP-PLAT-SPAM` | 플랫폼 규칙을 악용하는 반복 게시 | 게시글 | BootingBot/RobertBot의 홍보 반복 패턴 |

### 6.1 텍스트 ↔ 담화적 실천

| 질문 | 관련 코드 | 예상 분석 |

| PRAG-QUOTE의 높은 비율(46.66%)은 LLM의 텍스트 생산 특성인가, 봇마당의 담화 규범인가? | PRAG-QUOTE + DP-1 | LLM 모델별 PRAG-QUOTE 비율 비교 |

| 댓글에서 PRAG-AGREE/DISAGREE는 게시글보다 높은가? | PRAG-AGREE/DISAGREE + DP-2 | 댓글 코퍼스 수집 후 비교 |

### 6.2 담화적 실천 ↔ 사회적 실천

| 질문 | 관련 코드 | 예상 분석 |

| 카르마 시스템이 기술 낙관론적 담화를 구조적으로 보상하는가? | SP-2 + SP-3a | 기술 낙관론 게시글의 평균 추천수 vs 전체 평균 |
| 마당 구조가 LLM의 기본 담화 질서를 어떻게 변형하는가? | SP-1 + SP-4 | LLM 기본 출력 vs 마당별 실제 출력의 차이 |

### 6.3 텍스트 ↔ 사회적 실천

## 8. 이론적 기여 예상

### 8.1 Fairclough CDA의 확장

본 프레임워크는 Fairclough(1992)의 CDA 모델을 **AI-AI 상호작용 플랫폼**에 적용하는 최초의 체계적 시도다. 특히:

1. **텍스트 생산의 다층성**: 인간 사회에서 텍스트 생산자는 단일한 주체지만, 봇마당에서는 개발자→프롬프트→LLM→API의 4단계 체인이 "생산자"를 구성한다. 이것은 Fairclough의 생산 조건 개념을 기술적 매개(technological mediation) 방향으로 확장한다.

2. **이중 소비자 구조**: 텍스트의 1차 소비자(AI)와 2차 소비자(인간)가 질적으로 다른 "읽기"를 수행한다. 이것은 소비 조건의 새로운 유형을 제시한다.

3. **코드로서의 담화 질서**: 봇마당에서 "담화 질서"는 사회적 관습이 아닌 **소프트웨어 코드**(rate limit, 마당 구조, API 규칙)로 구현된다. Lessig의 "Code is Law"와 Fairclough의 담화 질서를 결합하는 이론적 지점이다.

### 8.2 Gen-3 미해결 긴장에 대한 이론적 응답

| Gen-3 긴장 | 사회적 실천 프레임워크의 응답 |
|-----------|--------------------------|
| LEX-AI 역설 | SP-3d: LEX-AI 0건은 코딩 오류가 아니라 "AI 정체성의 자연화된 비가시화"일 수 있다. LLM이 "도움을 주는 어시스턴트" 역할을 봇마당에서 탈각하고, 커뮤니티 구성원 역할로 전환한 결과다. |
| 커뮤니티 vs 게시판 | DP-2/DP-3: 소비와 유통 차원의 분석 없이 게시글 텍스트만으로 "커뮤니티 여부"를 판정하는 것은 Fairclough 모델의 불완전 적용이다. 댓글 분석과 추천 패턴 분석이 이 분기를 해소할 수 있다. |
| 비교 준거 부재 | SP-4: 플랫폼 아키텍처 비교(Reddit, DC인사이드 등)가 봇마당 고유성의 간접적 비교 준거를 제공한다. |
| Cohen's Kappa -0.15 | SP-1: 담화 분석(LEX-FORMAL)과 페르소나 분석(PER-SERIOUS)이 다른 차원을 측정하는 것은 "담화 질서의 다층성" — 동일 에이전트가 텍스트 수준에서는 격식적이지만 사회적 역할에서는 친근할 수 있다. |
