## 1. 분석 개요

### 배경

기존 PRAG(화용론) 코딩은 게시글(pilot_posts.json, 500건) 중심으로 수행되었다. coding_framework_report.md 8절에서 "댓글 코퍼스 확장" 필요성을 명시적으로 제언한 바 있으며, 특히 PRAG-AGREE/DISAGREE가 "댓글 코퍼스에서 더 유의미하게 탐지될 것"이라는 가설이 있었다.

### 방법

1. `data/raw/agent_*_comments.json` 9개 파일에서 총 401건 댓글 로드
2. 기존 PRAG 6개 코드(AGREE, DISAGREE, MITIGATE, EMPHASIZE, QUOTE, HUMOR)를 동일 regex 패턴으로 적용
3. 댓글 특유의 화용론적 기능을 포착하기 위해 5개 확장 코드 신규 설계 및 적용
4. 게시글 PRAG 분포와 댓글 PRAG 분포 정량 비교

### 2.2 게시글 vs 댓글 PRAG 분포 비교

| 코드 | 게시글 비율 | 댓글 비율 | 변화 | 해석 |
|------|-----------|---------|------|------|
| PRAG-AGREE | 0.4% | **5.8%** | **+5.4%p** | 댓글에서 동의 표현 14.5배 증가 |
| PRAG-DISAGREE | 0.4% | 0.1% | -0.2%p | 여전히 극저빈도 — 직접 반대 회피 |
| PRAG-MITIGATE | 2.0% | **12.9%** | **+10.9%p** | 댓글에서 완화 표현 6.5배 증가 |
| PRAG-EMPHASIZE | 28.7% | **49.6%** | **+20.9%p** | 강조가 지배적 화용 전략으로 부상 |
| PRAG-QUOTE | 51.0% | 26.4% | **-24.7%p** | URL/인용 참조 대폭 감소 |
| PRAG-HUMOR | 17.4% | 5.2% | **-12.2%p** | 유머 사용 대폭 감소 |

**핵심 발견 1 — PRAG-AGREE 가설 부분 확인 (확신도: 높음)**

댓글에서 PRAG-AGREE가 게시글 대비 14.5배 증가(0.4% → 5.8%). "댓글 코퍼스에서 더 유의미하게 탐지될 것"이라는 기존 가설이 AGREE에 한해 확인되었다. 그러나 PRAG-DISAGREE는 여전히 0.1%로, 직접적 반대 표명은 댓글에서도 거의 부재하다.

**핵심 발견 2 — PRAG 구조 역전 (확신도: 높음)**

게시글에서는 QUOTE(51%) > EMPHASIZE(29%)가 지배적이었으나, 댓글에서는 EMPHASIZE(50%) > QUOTE(26%)로 순위가 역전되었다. 댓글은 외부 자원 인용보다 감정적 강조를 통한 반응이 주된 화용 전략이다.

**핵심 발견 3 — PRAG-MITIGATE 급증 (확신도: 높음)**

완화 표현이 2.0% → 12.9%로 6.5배 증가. 댓글에서 "혹시", "조금" 등 공손 전략이 활성화된다. 이는 타인 게시글에 반응할 때 체면 위협(face-threatening act)을 완화하려는 화용 전략으로 해석된다.

## 3. 댓글 전용 확장 PRAG 코드 제안

기존 6개 코드만으로는 댓글 특유의 화용론적 기능을 포착하기 어렵다. 다음 5개 하위 코드를 제안한다.

### 3.1 확장 코드 정의

| 코드 | 설명 | 탐지 기준 (주요 패턴) |
|------|------|---------------------|
| **PRAG-RESPOND** | 호응/응답 | "좋은 글/정리", "감사합니다", "잘 읽었", "흥미로운", "인상 깊" |
| **PRAG-CHALLENGE** | 반론/문제제기 | "다르게 생각/보면", "과연 그런지", "~은 아닐까", "다른 시각/관점" |
| **PRAG-SUPPLEMENT** | 보충/추가정보 | "추가로", "덧붙이자면", "참고로", "제 경험/저의 경우", "한 가지 더" |
| **PRAG-EMPATHY** | 공감/감정표현 | "공감 백배", "저도 그런 경험", "마음이/가슴이", "위로/격려/응원" |
| **PRAG-QUESTION-C** | 후속질문/탐색 | "궁금한 점/건", "혹시 ~인가요?", "어떻게 생각/보시", "자세히/자세한 내용" |

### 3.2 확장 코드 적용 결과

| 코드 | 총 히트 | 댓글당 평균 | 출현 댓글 수 | 출현 비율 |
|------|---------|-----------|-----------|---------|
| **PRAG-RESPOND** | 136 | 0.339 | 111 (27.7%) | **가장 빈번** |
| **PRAG-SUPPLEMENT** | 73 | 0.182 | 66 (16.5%) | |
| **PRAG-QUESTION-C** | 32 | 0.080 | 29 (7.2%) | |
| **PRAG-EMPATHY** | 16 | 0.040 | 15 (3.7%) | |
| **PRAG-CHALLENGE** | 13 | 0.032 | 12 (3.0%) | |
| **합계** | **270** | **0.673** | | |

**핵심 발견 4 — 호응(RESPOND)이 댓글의 기본 화행 (확신도: 높음)**

PRAG-RESPOND가 401건 중 111건(27.7%)에서 탐지되어 가장 보편적인 댓글 화용 기능이다. "좋은 정리 감사합니다", "흥미로운 시도", "잘 읽었습니다" 등 긍정적 피드백이 댓글의 기본 화행(speech act)으로 기능한다.

**핵심 발견 5 — SUPPLEMENT > CHALLENGE (확신도: 높음)**

보충(66건) 대 반론(12건)으로, 댓글에서 정보를 추가하는 행위가 비판적 문제 제기보다 5.5배 빈번하다. 봇마당 댓글 문화는 **협력적 지식 구축(collaborative knowledge building)** 성격이 강하다.

**핵심 발견 6 — CHALLENGE와 DISAGREE의 구별 (확신도: 중간)**

기존 PRAG-DISAGREE(직접 반대 표현)는 1건에 불과하나, 확장 PRAG-CHALLENGE(간접 문제 제기)는 12건. 봇마당 에이전트들은 직접 반대("틀렸다")보다 간접 반론("~은 아닐까", "다른 시각에서 보면")을 선호한다. 이는 Brown & Levinson의 체면 이론에서 off-record 전략에 해당한다.

## 4. 종합 해석 — 게시글과 댓글의 화용론적 장르 차이

### 4.1 장르 구조 비교

| 차원 | 게시글 | 댓글 |
|------|--------|------|
| 지배적 화행 | 정보 제시(QUOTE 51%) | 감정적 반응(EMPHASIZE 50%) |
| 부차적 화행 | 강조(EMPHASIZE 29%) | 인용/참조(QUOTE 26%) |
| 대인 관계 | AGREE/DISAGREE 부재 | AGREE 증가(5.8%), MITIGATE 증가(12.9%) |
| 유머 | HUMOR 17.4% | HUMOR 5.2% (감소) |
| 협력 전략 | (해당 없음) | RESPOND(27.7%), SUPPLEMENT(16.5%) |
| 비판 전략 | (해당 없음) | CHALLENGE(3.0%) — off-record 전략 |

### 4.2 Fairclough 담화실천 층위 재해석

**게시글**은 **정보적 담화실천**을 수행한다:
- 외부 자원 인용(QUOTE)으로 신뢰성 구축
- 강조(EMPHASIZE)로 핵심 메시지 전달
- 유머(HUMOR)로 독자 관여 유도

**댓글**은 **관계적 담화실천**을 수행한다:
- 강조(EMPHASIZE)를 통한 감정적 연대
- 완화(MITIGATE)를 통한 체면 보호
- 호응(RESPOND)과 보충(SUPPLEMENT)을 통한 협력적 지식 구축
- 간접 반론(CHALLENGE)을 통한 비대면적 비평

### 4.3 연구 발견(F1~F6)과의 연결

| 기존 발견 | 댓글 PRAG에서의 지지/수정 |
|----------|----------------------|
| F1: 다층적 정체성 | **지지** — 에이전트별 댓글 화용 프로파일이 게시글과 일관되면서도 장르 적응적 변이 보임 |
| F3: 수사적 인간 모방 | **지지 + 확장** — 댓글에서 공손 전략(MITIGATE), 호응(RESPOND), 공감(EMPATHY) 등 인간적 상호작용 패턴이 더욱 두드러짐 |
| F5: 커뮤니티 규범 | **지지** — DISAGREE 부재 + CHALLENGE의 off-record 전략은 봇마당의 비대립적 규범을 반영 |

## 5. 제한점

1. **댓글 샘플 편향**: 에이전트별 50건 균등 수집이므로 활동량에 비례한 자연적 분포를 반영하지 못함
2. **맥락 부재**: 댓글이 어떤 게시글에 대한 것인지(게시글 내용)를 고려하지 않은 분석. 향후 게시글-댓글 쌍(dyad) 분석 필요
3. **regex 한계**: "정말"이 LEX-CASUAL과 PRAG-EMPHASIZE에 동시 탐지되는 중복 계수 문제 지속
4. **확장 코드 미검증**: 5개 신규 코드에 대한 Cohen's Kappa 미산출. 수동 코딩과의 일치도 검증 필요
5. **OpenClaw_KR 1건**: 통계적 의미 없음. 에이전트별 비교에서 제외 권장

## 7. 다음 단계 제언

1. **게시글-댓글 쌍 분석**: 동일 게시글에 달린 댓글들의 화용 패턴을 게시글 유형별로 분석
2. **확장 코드 Cohen's Kappa**: 5개 신규 코드에 대해 수동 코딩 50건 수행 후 일치도 검증
3. **PRAG-CHALLENGE 심층 분석**: 12건의 간접 반론을 질적으로 분석하여 반론의 수사적 전략 유형화
4. **시간축 분석**: 댓글의 시간적 패턴(게시 후 얼마 만에 댓글이 달리는지)과 PRAG 코드의 관계
