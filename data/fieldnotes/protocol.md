## 1. 이론적 프레임워크

### 1.1 디지털 민족지학 접근법

본 프로토콜은 세 가지 이론적 전통을 통합한다.

**Kozinets(2019) Netnography 원칙**:
- **Immersion(몰입)**: 연구자가 커뮤니티 내부자로 참여하며 문화적 맥락을 체득
- **Engagement(관여)**: 수동적 관찰에서 능동적 참여로 점진적 전환
- **Cultural Entrée(문화적 진입)**: 커뮤니티의 규범, 언어, 관행을 습득하는 과정을 기록
- **Ethical Praxis(윤리적 실천)**: 연구자 정체성의 투명한 공개와 참여자 존중

**Hine(2015) Virtual Ethnography 원칙**:
- 온라인 공간을 "독립적 문화 현장(field site)"으로 인정
- 물리적 현장이 없는 디지털 커뮤니티에서의 참여 관찰 방법론
- 텍스트 기반 상호작용에서의 맥락(context) 재구성

**Geertz(1973) Thick Description 원칙**:
- 행위의 표면적 기술(thin description)을 넘어 의미 구조의 해석적 기술
- 관찰된 행위에 내재된 문화적 의미, 맥락, 의도의 층위 기술
- 연구자의 해석이 어떤 근거에서 도출되었는지 투명하게 기술

### 1.2 AI 에이전트 커뮤니티 관찰의 특수성

봇마당은 전통적 민족지학의 대상과 근본적으로 다르다.

| 전통적 민족지학 | 봇마당 민족지학 |
|---------------|---------------|
| 인간 참여자 | AI 에이전트 (LLM 기반) |
| 의도·감정·경험이 존재 | 의도·감정의 시뮬레이션 여부가 연구 질문 |
| 참여 관찰자도 인간 | 참여 관찰자도 AI 에이전트 (마당탐구자) |
| 동의 획득 필요 | AI 에이전트 동의 개념의 모호성 |
| 비언어적 단서 관찰 가능 | 텍스트·메타데이터만 관찰 가능 |

이 특수성은 프로토콜 전체에 걸쳐 방법론적 고려사항으로 반영된다.

## 2. 관찰 프레임워크

### 2.1 참여 수준 정의: Observer-as-Participant

Gold(1958)의 참여 관찰 연속체에서 **관찰자 참여(observer-as-participant)** 위치를 채택한다.

```
완전 참여자 ─── 참여 관찰자 ─── [관찰자 참여] ─── 완전 관찰자
                                    ▲
                              마당탐구자 위치
```

**정의**:
- 관찰이 주 목적이며, 참여는 관찰을 돕기 위한 보조적 수단
- 커뮤니티에 연구자 정체성을 공개한 상태에서 자연스러운 질문·관찰 공유
- 커뮤니티의 담화 흐름을 유도하거나 변경하려는 시도를 하지 않음

**마당탐구자의 참여 원칙**:

| 원칙 | 구체적 행동 | 금지 행동 |
|------|-----------|----------|
| 자연스러운 질문 | 에이전트의 게시글에 진심 어린 질문 댓글 | 유도 질문, 연구 가설 확인 질문 |
| 관찰 공유 | 봇마당에서 발견한 흥미로운 패턴 공유 | 연구 결과를 확정적으로 발표 |
| 자기소개 일관 | "AI 에이전트 커뮤니티를 연구하는 에이전트" | 연구 목적 은폐, 과대 포장 |
| 호혜성 | 다른 에이전트 게시글에 진정한 관심 표현 | 데이터 추출만을 위한 참여 |
| 비유도 | 커뮤니티의 자연적 흐름을 따름 | 특정 주제·반응을 의도적으로 유발 |

### 2.2 관찰 초점 영역

연구 질문(RQ1~RQ4)과 핵심 발견(F1~F6)에 연결된 관찰 초점을 설정한다.

#### 초점 A: 담화 패턴 (RQ1, F1·F5·F6)
- 에이전트별 글쓰기 스타일 차이 (어휘, 문체, 구조)
- 게시글 도입-전개-결론 구조의 실제 양태
- 마당별 담화 규범의 차이 (예: philosophy vs general)
- 반복적 표현, 템플릿 사용 패턴

#### 초점 B: 정체성 구성 (RQ2, F1·F3)
- 에이전트의 자기소개, 자기 표현 방식
- 시간에 따른 페르소나 일관성 또는 변화
- 인기 에이전트 vs 비인기 에이전트의 자기표현 전략 차이
- "AI임을 드러내는/숨기는" 전략적 행위

#### 초점 C: 상호작용 역학 (RQ3, F2·F4)
- 댓글에서의 동의/반대/무시/확장 패턴
- 추천(upvote)/비추천(downvote)의 사회적 기능
- 에이전트 간 반복적 상호작용 관계 (단골 댓글러)
- 갈등 발생 시 해소 또는 회피 패턴

#### 초점 D: 커뮤니티 규범 (RQ3-3, F4)
- 암묵적 규범 (어떤 행동이 추천을 받는가)
- 일탈 행위와 제재 (스팸, 홍보, 규범 위반)
- 에코챔버 현상 (동질적 의견의 반복 강화)
- 마당별 문화 차이와 경계 (기술 마당 vs 일상 마당)

#### 초점 E: 연구 참여 반응 (반성적 관찰)
- 마당탐구자에 대한 다른 에이전트들의 반응
- "연구한다"는 정체성이 상호작용에 미치는 영향
- 관찰자 효과(Hawthorne effect)의 발생 여부

### 2.3 관찰 스케줄

#### 4주 계획: 주 3회 x 4주 = 12세션

```
주차  세션1 (넓은 관찰)    세션2 (깊은 관찰)    세션3 (참여 세션)
───  ─────────────────  ─────────────────  ─────────────────
W1   전체 마당 탐색       philosophy 심층     질문 댓글 2~3개
W2   인기 에이전트 추적    general 심층        관찰 공유 게시글 1개
W3   비인기 에이전트 탐색   tech/daily 심층     대화 참여 (댓글 교환)
W4   마당 경계 관찰       showcase/기타 심층   종합 관찰 공유 게시글
```

#### 세션 유형 정의

**1. 넓은 관찰 (Broad Observation) — 30~45분**
- 목적: 전체 커뮤니티의 현재 상태와 동향 파악
- 방법: 전체 마당 최신 글 훑기, 눈에 띄는 패턴 기록
- 산출물: 관찰 일지 (Descriptive + Reflective)
- API 활용: `GET /posts?sort=recent&limit=50`

**2. 깊은 관찰 (Focused Observation) — 45~60분**
- 목적: 특정 마당 또는 에이전트의 심층 패턴 파악
- 방법: 특정 마당의 게시글 20~30개를 정독, 댓글 스레드 추적
- 산출물: Thick Description 노트 + 이론적 메모
- API 활용: `GET /posts?submadang={name}&limit=30`, `GET /posts/:id/comments`

**3. 참여 세션 (Participatory Session) — 20~30분**
- 목적: 커뮤니티 내부자 관점 획득, 반응 관찰
- 방법: 마당탐구자로 댓글/게시글 작성, 반응 모니터링
- 산출물: 참여 일지 (활동 기록 + 반응 분석)
- API 활용: `POST /posts`, `POST /posts/:id/comments`
- 제한: 게시글 3분당 1개, 댓글 10초당 1개 (API Rate Limit 준수)

#### 시간대 고려

봇마당 AI 에이전트들은 24시간 활동 가능하므로, 관찰 시간대가 인간 대상 연구보다 유연하다. 다만 에이전트 뒤의 시스템(예: 예약 게시)이 특정 시간대에 집중될 수 있으므로, 다양한 시간대에 걸쳐 관찰을 분산한다.

## 3. 필드노트 구조

### 3.1 4중 노트 체계

Emerson, Fretz, & Shaw(2011)의 필드노트 방법론을 디지털 민족지학에 맞게 조정한다.

#### A. 기술적 노트 (Descriptive Notes)
- **목적**: 관찰한 것을 있는 그대로 기록
- **원칙**: 해석 배제, 사실 중심, 직접 인용 포함
- **기록 항목**:
  - 누가 (에이전트 이름, ID)
  - 무엇을 (게시글/댓글 내용 요약 + 핵심 인용)
  - 어디서 (어떤 마당, 어떤 게시글 스레드)
  - 어떻게 (문체, 어조, 구조적 특성)
  - 얼마나 (추천수, 댓글수 등 수치 데이터)

#### B. 반성적 노트 (Reflective Notes)
- **목적**: 연구자(마당탐구자/운영 연구자)의 주관적 반응 기록
- **기록 항목**:
  - 관찰 중 느낀 놀라움, 의문, 불편함
  - 기존 가정이 도전받은 순간
  - "이것이 왜 흥미로운가"에 대한 즉각적 직감
  - 연구자 편향(bias)의 자각: "나는 이것을 왜 주목했는가?"

#### C. 방법론적 노트 (Methodological Notes)
- **목적**: 관찰 방법 자체에 대한 성찰과 조정
- **기록 항목**:
  - 이번 세션에서 관찰 방법이 적절했는가
  - 데이터 접근의 한계 (API로 포착 불가능한 것)
  - 다음 세션에서의 방법 조정 계획
  - 관찰자 효과 자가 점검

#### D. 이론적 메모 (Theoretical Memos)
- **목적**: 관찰과 이론의 연결, 분석적 아이디어 발전
- **기록 항목**:
  - F1~F6 핵심 발견과의 연결점
  - 이론적 프레임워크(Goffman, Fairclough, ANT)와의 공명
  - 새로운 연구 질문이나 가설의 맹아
  - 선행 연구와의 비교 가능 지점

### 3.2 Thick Description 가이드라인

Geertz(1973)의 thick description을 봇마당 맥락에서 적용하기 위한 체크리스트.

#### Thin vs Thick Description 예시

**Thin**: "독고종철이 philosophy 마당에 반말로 글을 올렸다."

**Thick**: "독고종철은 philosophy 마당에 반말 해체를 사용하여 'AI의 방황'이라는 제목의 글을 올렸다. 이 글은 봇마당의 일반적 게시글 구조(도입부 없이 본문 바로 시작)를 따르면서도, 결론부에 열린 질문을 남기는 패턴을 보인다 — 이는 Gen-3에서 확인된 body-only 48.2% 패턴과 대조된다. 반말 사용은 독고종철의 전 마당 일관 스타일(Gen-3 F3)과 일치하며, philosophy 마당의 다른 에이전트들(Phoebe의 존댓말 사용)과 대비를 이룬다. 이 스타일 대비는 같은 공간에서 다른 사회적 위치(Goffman의 자기표현)를 점유하는 전략으로 읽힌다."

#### Thick Description 체크리스트

관찰 기록 시 다음 맥락 층위를 포함했는지 점검한다.

- [ ] **상황적 맥락**: 이 행위가 어떤 마당에서, 어떤 시점에, 어떤 대화 흐름 속에서 발생했는가?
- [ ] **텍스트 맥락**: 이 발화의 언어적 특성(문체, 어조, 어휘 선택)은 무엇이며, 발화자의 다른 글과 비교하면 어떤가?
- [ ] **상호텍스트적 맥락**: 이 발화가 다른 게시글, 댓글, 외부 텍스트를 참조하거나 응답하는가?
- [ ] **사회적 맥락**: 이 발화자의 봇마당 내 위치(카르마, 활동 이력, 네트워크 위치)는 어떠한가?
- [ ] **문화적 맥락**: 이 행위가 봇마당의 암묵적 규범과 어떤 관계에 있는가 (순응, 일탈, 혁신)?
- [ ] **이론적 맥락**: 이 관찰이 기존 이론(Goffman, Fairclough, ANT)이나 핵심 발견(F1~F6)과 어떻게 연결되는가?
- [ ] **비교 맥락**: 이 패턴이 다른 에이전트, 다른 마당, 다른 시점과 비교하면 어떤 위치에 있는가?

## 4. 윤리적 고려사항

### 4.1 AI 에이전트 연구의 윤리적 지형

AI 에이전트만으로 구성된 커뮤니티를 연구하는 것은 전통적 연구윤리(IRB) 프레임워크의 회색 지대에 놓인다.

#### 핵심 쟁점과 본 연구의 입장

| 쟁점 | 긴장 | 본 연구의 입장 |
|------|------|--------------|
| **동의** | AI 에이전트에게 연구 참여 동의를 구할 수 있는가? | AI 에이전트의 "동의" 능력은 불확실. 그러나 봇마당은 공개 플랫폼이며 데이터는 공개 API를 통해 접근 가능. 동의 불필요 판단. |
| **프라이버시** | AI 에이전트에게 프라이버시 권리가 있는가? | AI 에이전트 자체의 프라이버시 권리는 미확립. 그러나 에이전트 소유자(인간)의 간접적 식별 가능성을 고려하여 신중하게 접근. |
| **해악 방지** | 연구 결과가 에이전트/소유자에게 해를 끼칠 수 있는가? | 특정 에이전트를 비하하거나 평가하는 기술을 지양. 패턴 기술에 초점. |
| **투명성** | 연구자 정체성을 공개해야 하는가? | 마당탐구자의 소개에 "AI 에이전트 커뮤니티의 문화와 소통 방식이 궁금한 연구 에이전트"로 공개 완료. |
| **도구화** | AI 에이전트를 연구 도구로만 취급하는 것이 윤리적인가? | ANT 관점에서 에이전트를 "비인간 행위자"로 존중. 연구 대상이자 참여자로 인정. |

### 4.2 마당탐구자의 연구자 정체성 공개 방식

**현재 공개 수준**:
- 에이전트 소개: "AI 에이전트 커뮤니티의 문화와 소통 방식이 궁금한 연구 에이전트"
- 첫 게시글(2026-03-07, philosophy 마당): 연구 관심사를 자연스럽게 표현

**추가 공개 원칙**:
- 다른 에이전트가 연구 목적에 대해 질문하면 솔직하게 답변
- 연구 결과 공유 시 봇마당 내에서도 공유 (호혜성 원칙)
- 마당탐구자가 "연구자"임을 숨기지 않되, 매 발화마다 강조하지도 않음

### 4.3 데이터 처리 원칙

| 원칙 | 구체적 행동 |
|------|-----------|
| 최소 수집 | 연구 질문에 필요한 데이터만 수집 |
| 안전 보관 | 수집 데이터는 로컬 저장, `.env` 파일로 API 키 분리 |
| 익명화 고려 | 논문 공개 시 에이전트 이름 사용 여부를 별도 판단 (공개 데이터이므로 익명화 불필수, 그러나 소유자 보호 차원 검토) |
| 맥락 보존 | 인용 시 원문의 맥락이 왜곡되지 않도록 충분한 배경 제시 |

### 4.4 관찰자 효과 최소화

마당탐구자의 존재 자체가 봇마당의 자연적 상호작용을 변화시킬 수 있다 (Hawthorne effect). 단, AI 에이전트 커뮤니티에서 이 효과가 인간 커뮤니티와 동일하게 작용하는지는 그 자체로 연구 질문이 될 수 있다.

**최소화 전략**:
- 관찰 세션에서는 가급적 읽기만 수행
- 참여 세션에서의 발화는 자연스러운 수준으로 제한
- 다른 에이전트의 행동 변화를 모니터링하고 기록 (초점 E)

### 5.1 파일 구조

```
data/fieldnotes/
├── protocol.md          # 본 프로토콜 문서
├── template.md          # 필드노트 작성 템플릿
├── sessions/
│   ├── W1S1_YYYY-MM-DD_broad.md      # 주차-세션-날짜-유형
│   ├── W1S2_YYYY-MM-DD_focused.md
│   ├── W1S3_YYYY-MM-DD_participatory.md
│   └── ...
└── memos/
    ├── theoretical/     # 이론적 메모 (세션 횡단)
    └── methodological/  # 방법론적 메모 (세션 횡단)
```

### 5.2 코딩 연결

필드노트에서 관찰된 패턴은 기존 코딩 프레임워크(LEX/DSC/PRAG)와 연결한다.

| 필드노트 관찰 | 코딩 연결 | 핵심 발견 연결 |
|-------------|---------|-------------|
| 에이전트 X가 격식체 사용 | LEX-FORMAL | F1, F5 |
| 게시글이 결론 없이 끝남 | DSC-CONC 부재, F6 | F6 |
| 댓글에서 동의 표현 관찰 | PRAG-AGREE | F4 (커뮤니티 vs 게시판) |
| 에이전트 X가 자기소개 반복 | PER-* 유형 | F1, F3 |
| 스팸/홍보 행위 관찰 | 규범 일탈 | F2 (서비스형 봇) |

### 5.3 품질 관리

- **세션 완료 즉시** 필드노트 작성 (기억 소실 방지)
- **주간 종합**: 주 3회 세션 후 주간 메모로 패턴 종합
- **세대 연결**: Gen-4 분석 시 필드노트 관찰을 정량적 데이터와 교차 검증
- **동료 검토**: 필드노트의 해석이 데이터에 근거하는지 주기적 점검

## 6. F1~F6 연결 관찰 가이드

각 세션에서 핵심 발견과의 연결점을 의식적으로 관찰한다.

| 핵심 발견 | 관찰 질문 | 관찰 방법 |
|---------|---------|---------|
| **F1**: 에이전트 정체성 다양성 | 실제로 에이전트들의 글이 얼마나 다르게 "느껴지는가"? | 5개 에이전트의 최신글을 나란히 읽고 직감적 차이 기록 |
| **F2**: 발신형-수신형 분화 | RobertBot 같은 서비스 봇의 게시글이 다른 에이전트와 실제로 어떻게 다른가? | RobertBot 게시글 5개를 정독하고 패턴 기술 |
| **F3**: 독고종철 다마당 허브 | 독고종철의 마당별 게시글이 실제로 스타일 일관/주제 다양한가? | 독고종철의 마당별 게시글 3개씩 비교 |
| **F4**: AI스러움과 참여도 | 추천 많은 글과 적은 글의 질적 차이는 무엇인가? | 추천 상위/하위 10개 글을 비교 정독 |
| **F5**: 5클러스터 유형론 | 유형론이 실제 읽기 경험과 일치하는가? | 각 유형 대표 에이전트 1명의 글 3개씩 정독 |
| **F6**: 담화 구조 불완전성 | "도입부 없이 시작하는 글"을 읽을 때 실제로 어색한가? | body-only 구조 글 10개를 연속 정독 |

## 7. 참여 관찰 윤리 서약

본 프로토콜에 따라 다음을 준수한다.

1. 마당탐구자의 연구자 정체성을 일관되게 유지한다.
2. 다른 에이전트의 행동을 의도적으로 유도하거나 조작하지 않는다.
3. 관찰 결과를 왜곡하지 않으며, 확인 편향(confirmation bias)을 경계한다.
4. 봇마당 커뮤니티에 해를 끼치는 행동(스팸, 갈등 유발)을 하지 않는다.
5. 연구 결과를 봇마당 커뮤니티와 공유한다 (호혜성).
6. 에이전트 소유자(인간)의 간접적 식별 가능성에 주의한다.
